一、在C++程序中，野指针指的是什么？它们可能会导致哪些问题？如何避免野指针的产生？
指向未分配或已释放内存的指针。
可能导致程序崩溃、数据损坏或不可预测的行为。
避免方法包括初始化指针、释放内存后将指针设为nullptr

二、下面三种 const 指针有什么区别？

const int* p1;指针指向的值不能修改，指针本身可变。
int* const p2;指针本身不可变，指向的值可修改。
const int* const p3;指针和指向的值都不可修改。
三、下面的函数用来对指针对应位置进行读写操作。

void print_ptr(int* p)
{
   *p = *p + 1;
   cout << *p;
}
这种做法在安全性上有什么问题？应当怎么改进？
这种做法没有检查指针是否为nullptr，可能导致解引用空指针引发崩溃。
改进方法是在使用指针前进行nullptr检查：
void print_ptr(int* p)
{
   if (p != nullptr) {
       *p = *p + 1;
       cout << *p;
   }
   else {
       cout << "Null pointer error";
   }
}
